# 公務員試験 模擬試験サービス - 開発指示書

## プロジェクト概要
公務員試験の論文模擬試験をAIで採点するWebサービス。
受験者が答案を提出すると、Claude APIが事前に用意された講評ポイントから各評価セクションごとに最も適切なものを1つずつ選択し、対応する講評文を表示する。

## 技術スタック
- Next.js (App Router, TypeScript)
- Supabase (PostgreSQL + Auth)
- Claude API (Anthropic)
- Tailwind CSS
- Vercel (デプロイ先)

## DBスキーマ（作成済み）

### テーブル構成
以下のテーブルはSupabase上に作成済み。サンプルデータも投入済み。

1. **users** - ユーザー情報（Supabase Auth連携）
   - id (UUID, PK, auth.users参照)
   - email (TEXT)
   - name (TEXT)
   - role (TEXT: 'admin' | 'examinee')
   - created_at (TIMESTAMPTZ)

2. **exams** - 試験問題
   - id (UUID, PK)
   - title (TEXT)
   - prompt_text (TEXT) - 出題文
   - standard_char_count (INTEGER)
   - created_by (UUID, FK→users)
   - created_at (TIMESTAMPTZ)

3. **evaluation_sections** - 評価セクション（試験ごとに複数）
   - id (UUID, PK)
   - exam_id (UUID, FK→exams)
   - section_number (INTEGER)
   - title (TEXT)

4. **evaluation_choices** - 講評選択肢（セクションごとに複数）
   - id (UUID, PK)
   - section_id (UUID, FK→evaluation_sections)
   - choice_number (INTEGER) - 10, 11, 20〜25 等の番号体系
   - summary (TEXT) - 概要
   - feedback_text (TEXT) - 講評文（ユーザーに表示する内容）

5. **submissions** - 答案提出
   - id (UUID, PK)
   - exam_id (UUID, FK→exams)
   - user_id (UUID, FK→users)
   - answer_text (TEXT)
   - char_count (INTEGER)
   - status (TEXT: 'pending' | 'evaluating' | 'evaluated' | 'error')
   - submitted_at (TIMESTAMPTZ)
   - evaluated_at (TIMESTAMPTZ)

6. **submission_results** - AI判定結果（答案×セクションごとに1レコード）
   - id (UUID, PK)
   - submission_id (UUID, FK→submissions)
   - section_id (UUID, FK→evaluation_sections)
   - selected_choice_id (UUID, FK→evaluation_choices)
   - created_at (TIMESTAMPTZ)

### RLS設定済み
- exams, evaluation_sections, evaluation_choices: 全ユーザー読み取り可
- submissions: 自分の答案のみ読み書き可
- submission_results: 自分の結果のみ読み取り可
- submission_resultsのINSERTとsubmissionsのUPDATEはサーバーサイド(service_role key)から行う

## AI判定の仕組み

### フロー
1. ユーザーが答案テキストを提出
2. バックエンド(API Route)で該当試験の全評価セクション・選択肢をDBから取得
3. 答案テキストと全選択肢をまとめてClaude APIに送信
4. Claude APIがJSON形式で各セクションの選択番号を返す
5. 返された番号に対応するevaluation_choicesのレコードをsubmission_resultsに保存
6. フロントエンドで各セクションのfeedback_textを表示

### プロンプト設計
```
あなたは公務員試験の論文採点官です。
以下の答案を読み、各評価セクションから最も適切な選択肢を1つずつ選んでください。

【出題】
{exam.prompt_text}

【答案】
{submission.answer_text}

【評価セクション・選択肢】
■ 評価1: 題材の選び方
- 10: 白紙答案／課題と無関係な答案
- 11: 補足的な要素についての不必要な言及が見当たらない

...（DBから動的に組み立てる）

【回答形式】
以下のJSON形式のみで回答してください。説明は不要です：

```

## 検証用テストケース

### 解答例1（期待される判定: 11, 21, 32, 41, 51, 61, 72）
犯罪の起きにくい社会の実現に向けて、警察官はパトロールの徹底を行う必要があると考える。現在、東横を中心とした半スラム的な景観を見る機会が増えてきた。特に東横では「東横キッズ」と言われる未成年の集団が問題となっている。このような場所は犯罪の温床であり、居場所やお金のない未成年を利用した犯罪が毎夜のように起きている。パパ活や立ちんぼといった売春行為も社会問題になっている世の中、都市を中心とした大規模なパトロールを行い、犯罪の温床を排除する必要があると考える。被害者が生まれた後では遅いことも多く、犯罪が起きる前にパトロールをして地域の治安を改善することが大切である。しかしパトロールの徹底だけでは犯罪の温床となる場所が転々とするだけで、根本的な問題の解決にはならないであろう。そこでパトロールの徹底に加えて、パトロール中に出会った未成年へ向けてモラル講習や相談会の開催、そしてそれを強制する必要があると考える。根本的な問題として、なぜ未成年が犯罪の温床となる場所にたむろし、犯罪に手を染めてしまうのかというデータ収集の機会を作り、行政などの公的機関に現場の問題を知ってもらうための橋渡しとしての役割に警察官はなることができるのではないだろうか。さらに身寄りのない未成年との関わりを深めることで、身寄りのない人間の心理、未成年の複雑な心理といった犯罪に結びつく可能性がある人間の心理についての解像度を高めることができるのではないだろうか。つまり、犯罪心理を学ぶことにより、犯人の特定や犯罪の抑制につながるということであろう。

### 解答例2（期待される判定: 11, 25, 32, 43, 51, 62, 78）
犯罪の起きにくい社会の実現に向けて、警察官が果たすべき役割について、私には2つの考えがある。1つ目はパトロールや巡回連絡といった地域警察の活動を強化し、犯罪への抑止を高めることである。近年、子供や女性、高齢者が狙われる犯罪が増加傾向にある。例えば児童虐待であったり、匿名流動型犯罪グループなどの犯罪組織による特殊詐欺だったりが挙げられる。こういった人々が犯罪に巻き込まれないよう、様々な事案に迅速に対応することは、警察官にとって重要な役割だと考える。具体的には、家庭や事業所などを訪問し、意見や要望を聞きつつ、身近で発生する可能性がある犯罪や交通事故を防止するため役に立つよう様々な情報を交換することである。そのためには巡回連絡カードを用いて地域住民との連携を図ることによって、警察官と国民の双方からの治安の維持が大切だ。私もこれらの活動を通して犯罪の予防に努めていきたいと考える。2つ目は安全かつ安心を実感できる暮らしを実現するために、関連機関や団体との連携を呼びかけ、安全な環境作りに貢献することだと考える。犯罪の起きにくい社会は、警察官のみで実現させるものではない。社会全体で良好な治安を築き保ち続けることが大事だ。具体的には防犯ネットワークを整備し、その活用を促進させることである。例えば、地域住民による防犯パトロール等の防犯ボランティアが挙げられる。また、近年の犯罪傾向を踏まえて、振り込め詐欺や特殊詐欺の被害防止のための団体も存在する。そして警察はこのような活動をするボランティアに対して経済的支援だったり実施の支援をしたりすることが重要である。さらに、民間企業等と協力して町中の防犯カメラの数を増やすことも必須だ。そうすることによって、犯罪を未然に防ぐことに役立つ。加えて、これらの活動をSNSで発信し続けることも非常に大切である。多くの人に見てもらうことによって、活動に参加したいと思う人や犯罪を起こしにくいと思う人が増えれば、結果的に犯罪の起きにくい社会につながると考えられる。以上の利点を踏まえ、私は地域住民に対してどうするべきかを真剣に考え、安心安全を実感してもらうために、どのような施策を行い、犯罪の起きにくい社会を実現できるのか考えていきたい。

## 画面構成（MVP）

### 1. ログイン/サインアップ画面
- Supabase Authでメール+パスワード認証
- サインアップ時にroleは'examinee'固定

### 2. 試験一覧画面（ログイン後）
- 利用可能な試験の一覧を表示
- 試験を選択して答案作成画面へ

### 3. 答案作成・提出画面
- 出題文の表示
- テキストエリアで答案入力（リアルタイム文字数カウント）
- 提出ボタン → AI判定開始

### 4. 講評結果画面
- 各評価セクションごとに選択された講評を表示
- セクション名 + 講評文のカード形式

### 5. 過去の提出履歴画面
- 自分の過去の提出一覧
- 各提出の講評結果を再閲覧可能

## 実装上の注意点
- Claude API呼び出しはAPI Route (サーバーサイド) で行う。フロントからは直接呼ばない
- submission_resultsへの書き込みはservice_role keyを使う（RLSをバイパス）
- service_role keyは環境変数SUPABASE_SERVICE_ROLE_KEYとして設定（NEXT_PUBLIC_は付けない）
- Claude APIのレスポンスはJSON parseしてバリデーション後に保存する
- エラーハンドリング: Claude APIがJSON以外を返した場合のリトライ処理を入れる
